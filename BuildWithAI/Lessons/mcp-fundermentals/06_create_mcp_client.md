# Creating an MCP Client

## Overview

With your MCP server ready, it's now time to create an **MCP client** inside your chatbot. This will let the chatbot communicate with the server and get access to tool definitions and results.

In the previous lesson, you created an MCP research server that exposes 2 tools. In this lesson, you'll make the chatbot communicate with the server through an MCP client, making the chatbot **MCP-compatible**.

---

## Understanding MCP Clients

Recall the MCP architecture from Lesson 03:

```
┌─────────────────────────────────────────────────────────────────┐
│                          Host                                   │
│                      (Your Chatbot)                             │
│                                                                 │
│                      ┌─────────────┐                            │
│                      │ MCP Client  │                            │
│                      └──────┬──────┘                            │
└─────────────────────────────┼───────────────────────────────────┘
                              │
                              │ stdio transport
                              │
                        ┌─────▼──────┐
                        │ MCP Server │
                        │ (research) │
                        └────────────┘
```

**What you're building:**
- **Host**: Your chatbot application
- **MCP Client**: Connects to the research server, lists tools, executes tools
- **Transport**: Standard I/O (subprocess)

## The MCP Client Approach

With MCP, we shift responsibilities:

| Responsibility | Before MCP (Lesson 04) | After MCP (Lesson 06) |
|----------------|------------------------|------------------------|
| **Tool Definitions** | In chatbot code | In MCP server |
| **Tool Execution** | In chatbot code | In MCP server |
| **Tool Schemas** | Manually written | Auto-generated by FastMCP |
| **Chatbot's Job** | Everything | Query user, call LLM, manage conversation |

**What's new:**
- `ClientSession` — Manages the connection to an MCP server
- `stdio_client` — Creates a client using Standard I/O transport
- `StdioServerParameters` — Configuration for launching the server
- `asyncio` — Async/await for non-blocking operations
- `nest_asyncio` — Compatibility for different operating systems

## Step 1: Configure Server Parameters

Tell the client how to launch and connect to the server:

```python
# Define server parameters
server_params = StdioServerParameters(
    command="uv",
    args=["run", "research_server.py"],
    env=None  # Pass environment variables if needed
)
```

**What's happening:**
- `command="uv"` — The command to run
- `args=["run", "research_server.py"]` — Arguments (runs your server file)
- `env=None` — Optional environment variables

This is the same command you used in the Inspector: `uv run research_server.py`

**State we're tracking:**
- `anthropic_client` — Claude API client
- `session` — MCP client session (once connected)
- `tools` — Tool definitions from the server

## Step 3: Connect to Server and List Tools

Create an async function to establish connection:

```python
async def connect_to_server_and_run(self):
    """Connect to MCP server and start chat loop."""
    
    # Establish connection to MCP server
    async with stdio_client(server_params) as (read, write):
        # Create client session
        async with ClientSession(read, write) as session:
            # Save session for later use
            self.session = session
            
            # Initialize the session (handshake)
            await session.initialize()
            
            # List available tools from server
            tools_response = await session.list_tools()
            
            # Convert MCP tool format to Anthropic format
            self.tools = [
                {
                    "name": tool.name,
                    "description": tool.description,
                    "input_schema": tool.inputSchema
                }
                for tool in tools_response.tools
            ]
            
            print(f"Connected to server with {len(self.tools)} tools: {[t['name'] for t in self.tools]}")
            
            # Start the chat loop
            await self.chat_loop()
```

**What's happening:**

1. **`stdio_client(server_params)`** — Launches server as subprocess, gets read/write streams
2. **`ClientSession(read, write)`** — Creates session to communicate with server
3. **`session.initialize()`** — Performs initialization handshake (remember from Lesson 03!)
4. **`session.list_tools()`** — Sends `ListToolsRequest` to server
5. **Convert tool format** — Transform MCP tools to Anthropic's expected format
6. **Start chat loop** — Begin accepting user queries

## Step 4: Process Queries with Tool Execution

Update the query processing to use the MCP server:

```python
async def process_query(self, query):
    """Process a single query with Claude and MCP tools."""
    messages = [{'role': 'user', 'content': query}]
    
    processing = True
    
    while processing:
        # Call Claude with MCP tools
        response = self.anthropic_client.messages.create(
            max_tokens=2024,
            model='claude-3-7-sonnet-20250219',
            tools=self.tools,  # Tools from MCP server
            messages=messages
        )
        
        assistant_content = []
        
        for content in response.content:
            if content.type == 'text':
                print(content.text)
                assistant_content.append(content)
                
            elif content.type == 'tool_use':
                assistant_content.append(content)
                
                # Add assistant message
                messages.append({
                    'role': 'assistant',
                    'content': assistant_content
                })
                
                # Execute tool through MCP server
                tool_name = content.name
                tool_args = content.input
                tool_id = content.id
                
                print(f"\nCalling tool '{tool_name}' with args {tool_args}")
                
                # This is the key difference: call tool through MCP
                result = await self.session.call_tool(tool_name, tool_args)
                
                # Extract text content from result
                tool_result_content = result.content[0].text if result.content else ""
                
                # Add tool result to messages
                messages.append({
                    "role": "user",
                    "content": [{
                        "type": "tool_result",
                        "tool_use_id": tool_id,
                        "content": tool_result_content
                    }]
                })
                
                # Continue processing
                break
        
        # Check if we're done
        if len(response.content) == 1 and response.content[0].type == "text":
            processing = False
```

**Key difference from Lesson 04:**

**Before (Lesson 04):**
```python
# Execute tool locally
result = execute_tool(tool_name, tool_args)
```

**After (Lesson 06):**
```python
# Execute tool through MCP server
result = await self.session.call_tool(tool_name, tool_args)
```

The MCP client sends a `CallToolRequest` to the server, which executes the tool and returns the result!

## Complete MCP Chatbot File

Save this as `mcp_chatbot.py`:

```python
#!/usr/bin/env python3
"""MCP-enabled chatbot that connects to research server."""

from mcp.client.session import ClientSession
from mcp.client.stdio import StdioServerParameters, stdio_client

import asyncio
import nest_asyncio
import anthropic
from dotenv import load_dotenv
import os

nest_asyncio.apply()
load_dotenv()


class MCPChatbot:
    def __init__(self):
        """Initialize the MCP chatbot."""
        self.anthropic_client = anthropic.Anthropic(
            api_key=os.environ.get("ANTHROPIC_API_KEY")
        )
        self.session = None
        self.tools = []

    async def connect_to_server_and_run(self):
        """Connect to MCP server and start chat loop."""
        server_params = StdioServerParameters(
            command="uv",
            args=["run", "research_server.py"],
            env=None
        )
        
        async with stdio_client(server_params) as (read, write):
            async with ClientSession(read, write) as session:
                self.session = session
                
                # Initialize session
                await session.initialize()
                
                # List tools
                tools_response = await session.list_tools()
                self.tools = [
                    {
                        "name": tool.name,
                        "description": tool.description,
                        "input_schema": tool.inputSchema
                    }
                    for tool in tools_response.tools
                ]
                
                print(f"\n✓ Connected to server with {len(self.tools)} tools:")
                for tool in self.tools:
                    print(f"  - {tool['name']}")
                
                # Start chat loop
                await self.chat_loop()

    async def process_query(self, query):
        """Process a single query with Claude and MCP tools."""
        messages = [{'role': 'user', 'content': query}]
        
        processing = True
        
        while processing:
            response = self.anthropic_client.messages.create(
                max_tokens=2024,
                model='claude-3-7-sonnet-20250219',
                tools=self.tools,
                messages=messages
            )
            
            assistant_content = []
            
            for content in response.content:
                if content.type == 'text':
                    print(content.text)
                    assistant_content.append(content)
                    
                elif content.type == 'tool_use':
                    assistant_content.append(content)
                    
                    messages.append({
                        'role': 'assistant',
                        'content': assistant_content
                    })
                    
                    tool_name = content.name
                    tool_args = content.input
                    tool_id = content.id
                    
                    print(f"\n[Calling tool '{tool_name}' via MCP server...]")
                    
                    # Execute tool through MCP
                    result = await self.session.call_tool(tool_name, tool_args)
                    tool_result_content = result.content[0].text if result.content else ""
                    
                    messages.append({
                        "role": "user",
                        "content": [{
                            "type": "tool_result",
                            "tool_use_id": tool_id,
                            "content": tool_result_content
                        }]
                    })
                    
                    break
            
            if len(response.content) == 1 and response.content[0].type == "text":
                processing = False

    async def chat_loop(self):
        """Run the interactive chat loop."""
        print("\nType your queries or 'quit' to exit.\n")
        
        while True:
            try:
                query = input("Query: ").strip()
                
                if query.lower() == 'quit':
                    print("Goodbye!")
                    break
                
                await self.process_query(query)
                print()
                
            except Exception as e:
                print(f"\nError: {str(e)}\n")


def main():
    """Main entry point."""
    chatbot = MCPChatbot()
    asyncio.run(chatbot.connect_to_server_and_run())


if __name__ == "__main__":
    main()
```

## What's Happening Under the Hood

### Connection Flow

```
1. Chatbot launches
   ↓
2. stdio_client creates subprocess for server
   ↓
3. ClientSession established with read/write streams
   ↓
4. Initialize handshake
   ↓
5. List tools from server
   ↓
6. Convert tools to Anthropic format
   ↓
7. Ready for user queries
```

### Tool Execution Flow

```
User query
   ↓
Claude decides to use tool
   ↓
Chatbot: session.call_tool(name, args)
   ↓
MCP Client → Server (CallToolRequest)
   ↓
Server executes function
   ↓
Server → Client (tool result)
   ↓
Chatbot passes result back to Claude
   ↓
Claude generates final response
```

## Key Concepts

### MCP Protocol Messages

During your session, you'll see these MCP protocol messages in the logs:

| Message | Purpose |
|---------|---------|
| `InitializeRequest` | Client asks to connect |
| `InitializeResponse` | Server confirms connection |
| `ListToolsRequest` | Client asks for available tools |
| `ListToolsResponse` | Server provides tool definitions |
| `CallToolRequest` | Client asks to execute a tool |
| `CallToolResponse` | Server returns tool result |

### Async/Await

The code uses `async`/`await` because:
- I/O operations (subprocess communication) shouldn't block
- Multiple operations can happen concurrently
- Better performance for real applications

```python
# Async function
async def connect_to_server_and_run(self):
    # Wait for I/O operation
    await session.initialize()
    
    # Wait for another I/O operation
    tools_response = await session.list_tools()
```

## Comparison: Before and After MCP

### Architecture Comparison

**Before MCP (Lesson 04):**
```
┌─────────────────────────────────────┐
│          Chatbot                    │
│                                     │
│  - Tool definitions                 │
│  - Tool schemas (manual)            │
│  - Tool execution code              │
│  - Chat loop                        │
│  - Claude API calls                 │
└─────────────────────────────────────┘
```

**After MCP (Lesson 06):**
```
┌─────────────────────────────────────┐
│          Chatbot (Host)             │
│                                     │
│  - Chat loop                        │
│  - Claude API calls                 │
│  - MCP Client                       │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│       MCP Server                    │
│                                     │
│  - Tool definitions                 │
│  - Tool schemas (auto-generated)    │
│  - Tool execution code              │
└─────────────────────────────────────┘
```

### Benefits of MCP Approach

| Aspect | Before MCP | After MCP |
|--------|------------|-----------|
| **Separation of Concerns** | Everything in one file | Chatbot vs. Server |
| **Reusability** | Tools tied to this chatbot | Server reusable across apps |
| **Maintenance** | Update chatbot for tool changes | Update server, chatbot unchanged |
| **Testing** | Need full chatbot to test tools | Test server independently (Inspector) |
| **Schema Management** | Manual | Automatic |

## Troubleshooting

### Issue: "Connection refused"

**Solution:**
- Verify `research_server.py` exists in the same directory
- Check that the server file has no syntax errors
- Try running the server directly: `uv run research_server.py` (should hang waiting for input)

### Issue: "No tools found"

**Solution:**
- Ensure `@mcp.tool()` decorators are on your functions
- Check that `mcp.run(transport="stdio")` is in the server file

### Issue: "Async event loop error"

**Solution:**
- Make sure `nest_asyncio.apply()` is called at the top of the file
- On some systems, you may need to install `nest-asyncio` separately
